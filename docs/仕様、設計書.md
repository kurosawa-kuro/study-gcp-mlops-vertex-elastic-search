## 2. システムアーキテクチャ

```
[Cloud Run Job: hello-elastic-job]
   ・Python 3.11 + elasticsearch + google-cloud-secret-manager
   ・2つの動作モード:
     - ingest: クリーンアップ → data/*.json投入 → 確認検索（データ残す）
     - search: 既存インデックスにキーワード検索
   ・環境変数 GCP_PROJECT / ES_SECRET_NAME を注入
         │
         ├── Secret Manager読み取り
         │       ↓
         │   [Secret Manager: hello-elastic-api-key]
         │       └── JSON: {cloud_url, username, password}
         │              ← ec_deploymentから自動取得
         │
         └── ES接続
                 ↓
             [Elastic Cloud: hello-elastic]
                 ├── Elasticsearch 9.3.2（検索・格納）
                 └── Kibana（管理UI）

         ↑ イメージ取得
[Artifact Registry: hello-elastic-repo]
   ・Docker形式
   ・イメージ: hello-elastic-job:latest

[GitHub Actions]
   └── terraform.yml: push(terraform/**) → plan → apply
```

## 4. Terraform設計

### プロバイダ

| 項目 | 値 |
|------|-----|
| Terraform | >= 1.0 |
| ec provider | ~> 0.12 |
| google provider | ~> 5.0 |
| 認証方式（ec） | Elastic Cloud Org APIキー（`TF_VAR_elastic_cloud_api_key`） |
| 認証方式（google） | デフォルト認証（ADC） |

### 変数（.envから`TF_VAR_`経由で注入）

| 変数名 | .envキー | 用途 |
|--------|---------|------|
| project_id | PROJECT_ID | GCPプロジェクトID |
| region | REGION | GCPリージョン |
| deployment_name | DEPLOYMENT_NAME | Elastic Cloudデプロイメント名 |
| job_name | JOB_NAME | Cloud Run Job名 |
| repo_name | REPO_NAME | Artifact Registryリポジトリ名 |
| secret_name | SECRET_NAME | Secret Managerシークレット名 |
| elastic_cloud_api_key | ELASTIC_CLOUD_API_KEY | Elastic Cloud Org APIキー（sensitive） |

### Secret Manager設計

手動APIキー管理を廃止。`ec_deployment`の出力からJSON自動格納:

```json
{
  "cloud_url": "<ec_deployment.hello.elasticsearch.https_endpoint>",
  "username":  "<ec_deployment.hello.elasticsearch_username>",
  "password":  "<ec_deployment.hello.elasticsearch_password>"
}
```

### サービスアカウント

| アカウント | 用途 |
|-----------|------|
| terraform@mlops-dev-a.iam.gserviceaccount.com | Terraformリソース操作（roles/editor, roles/run.admin） |
| {PROJECT_NUMBER}-compute@developer.gserviceaccount.com | Cloud Run Job実行（roles/secretmanager.secretAccessor） |

## 5. アプリケーション設計

### Cloud Run Job

| 項目 | 値 |
|------|-----|
| ベースイメージ | python:3.11-slim |
| 依存ライブラリ | elasticsearch 8.13.0, google-cloud-secret-manager 2.22.0 |
| エントリポイント | python main.py |
| イメージURI | asia-northeast1-docker.pkg.dev/mlops-dev-a/hello-elastic-repo/hello-elastic-job:latest |

### 環境変数

| 変数名 | 値 | 設定元 |
|--------|-----|--------|
| GCP_PROJECT | mlops-dev-a | Terraform (main.tf) |
| ES_SECRET_NAME | hello-elastic-api-key | Terraform (main.tf) |
| SEARCH_KEYWORD | (任意) | `gcloud run jobs execute --update-env-vars` で実行時上書き |

### 動作モード

| モード | 条件 | 処理フロー |
|--------|------|-----------|
| ingest | `SEARCH_KEYWORD` 未設定 | cleanup → info → data/*.json投入 → match_all確認 → **データ残す** |
| search | `SEARCH_KEYWORD` 設定済 | 既存インデックスに multi_match 検索 |

### インデックス設計

| 項目 | 値 |
|------|-----|
| インデックス名 | hello |
| スキーマ | 動的マッピング（JSONフィールドから自動生成） |
| データソース | src/data/*.json（配列形式 or 単体オブジェクト） |

## 7. デプロイ設計

### deploy-all フロー（3段階・冪等）

```
make deploy-all
  ├── 1. tf-apply-infra    Elastic Cloud + Artifact Registry + Secret Manager + IAM
  ├── 2. push              Docker build → Artifact Registry push
  └── 3. tf-apply          Cloud Run Job 作成（全リソース適用）
```

### CI/CD（GitHub Actions）

```
terraform.yml: push(terraform/**) or PR → plan表示、main push → apply自動
```

### コード変更時のデプロイ

```
make push → make ingest（tf-apply 不要。:latest タグ参照のため）
```
