## 1. 前提条件

| 項目 | 確認コマンド |
|------|-------------|
| GCP CLI | `gcloud --version` |
| Terraform | `terraform --version` |
| Docker | `docker --version` |
| GCP認証済み | `gcloud auth list` |
| Docker認証済み | `make gcp-setup-docker`（初回のみ） |
| `.env` 設定済み | `.env`（プロジェクトルート） |

### .env の設定項目

```bash
# GCP
PROJECT_ID=mlops-dev-a
REGION=asia-northeast1

# Elastic Cloud
DEPLOYMENT_NAME=hello-elastic
ELASTIC_CLOUD_API_KEY=<Org APIキー>  # https://cloud.elastic.co/ → Organization → API keys

# Cloud Run Job
JOB_NAME=hello-elastic-job
REPO_NAME=hello-elastic-repo
SECRET_NAME=hello-elastic-api-key
```

> `ELASTIC_CLOUD_API_KEY` は Elastic Cloud の **Org レベル** APIキー（Terraform provider認証用）。
> ES接続用の認証情報（URL/username/password）は Terraform が `ec_deployment` から自動取得し Secret Manager にJSON格納するため、手動管理不要。
> `GCP_PROJECT`/`ES_SECRET_NAME` はMakefileが `PROJECT_ID`/`SECRET_NAME` から自動導出する。

## 2. 全構築: `make deploy-all`

```bash
make deploy-all
```

### 実行フロー

```
make deploy-all
  │
  ├── 1. make tf-apply-infra    ← Cloud Run Job以外を先行作成
  │       ├── ec_deployment（Elastic Cloud デプロイメント）
  │       ├── google_artifact_registry_repository
  │       ├── google_secret_manager_secret
  │       ├── google_secret_manager_secret_version  ← 接続情報JSONを自動格納
  │       └── google_secret_manager_secret_iam_member
  │
  ├── 2. make push              ← Dockerイメージをビルド & Artifact Registryへpush
  │       ├── docker build -t <IMAGE_URI> src/
  │       └── docker push <IMAGE_URI>
  │
  └── 3. make tf-apply          ← Cloud Run Job含む全リソースを適用
          └── google_cloud_run_v2_job（イメージ参照 + GCP_PROJECT/ES_SECRET_NAME注入）
```

### なぜ3段階か

1. **tf-apply-infra**: Artifact Registryを先に作らないとDockerイメージをpushできない。Elastic CloudとSecret Managerもここで作成
2. **push**: イメージがレジストリにないとCloud Run Jobの作成が失敗する
3. **tf-apply**: イメージが存在する状態でCloud Run Jobを作成・更新

### deploy-all 完了後

```bash
make ingest                  # データ投入
make search Q=Elasticsearch  # キーワード検索
```

## 3. 全削除: `make destroy-all`

```bash
make destroy-all
```

### 実行フロー

```
make destroy-all
  │
  ├── 1. make tf-destroy    ← 全リソースを削除
  │       ├── Cloud Run Job
  │       ├── Secret Manager（secret + version + IAM）
  │       ├── Artifact Registry
  │       └── Elastic Cloud デプロイメント
  │
  └── 2. make clean          ← ローカルDockerイメージを削除
```

> Elastic Cloud デプロイメントも `terraform destroy` で削除される。**課金が即座に停止する。**

### 再構築

```bash
make deploy-all    # 全て再作成（5分程度）
```

> .envの変更は不要。URLやパスワードはTerraformが毎回新規生成しSecret Managerに格納する。

## 4. 日常オペレーション

全てプロジェクトルートで実行する。

### データ投入: `make ingest`

```bash
make ingest
```

`src/data/*.json` を全て読み込み、ESに投入する。**データは残る**（searchモードで利用するため）。

期待されるログ出力:

```
=== 1. cleanup ===
index 'hello' does not exist (skip)
=== 2. info ===
cluster_name: xxx
version: 9.3.2
=== 3. index documents (hello) ===
loaded: sample.json (3 docs)
  [1/3] created: Elasticsearch入門
  [2/3] created: Terraform入門
  [3/3] created: Cloud Run概要
=== 4. search (match_all) ===
total hits: 3
  -> {'title': 'Elasticsearch入門', ...}
  -> {'title': 'Terraform入門', ...}
  -> {'title': 'Cloud Run概要', ...}
done. index 'hello' retained for search testing.
```

### キーワード検索: `make search Q=...`

```bash
make search Q=Elasticsearch
make search Q=Cloud
```

投入済みインデックスに対して `multi_match` 検索（全フィールド対象）を実行する。

期待されるログ出力:

```
=== search: "Elasticsearch" ===
total hits: 1
  score=1.0227  {'title': 'Elasticsearch入門', ...}
```

> インデックスが存在しない場合はエラー。先に `make ingest` を実行する。

### ログ確認

```bash
make logs    # 最新executionのログを表示
```

> `make ingest` / `make search` は実行後に自動でログを表示する。

### 投入データの追加

`src/data/` にJSONファイルを追加し、`make push` で反映する。

```bash
vi src/data/new_data.json    # JSONファイルを追加（配列形式 or 単体オブジェクト）
make push                    # イメージ再ビルド & push
make ingest                  # 再投入
```

### コード変更後の再デプロイ

```bash
make push       # イメージ再ビルド & push
make ingest     # 新イメージでJob実行
```

> Cloud Run Jobは `:latest` タグを参照しているため、pushだけで次回実行時に新イメージが使われる。`tf-apply` は不要。

## 5. ローカル開発

### ローカル実行（Secret Manager経由）

```bash
make install    # 依存パッケージインストール
make run        # python3 src/main.py（Makefileが.envをexportして実行）
```

> ローカル実行には `gcloud auth application-default login` でADC認証が必要。

### Docker実行

```bash
make docker-run    # docker build → docker run --env-file .env
```

## 7. トラブルシューティング

### Artifact Registry push失敗

```bash
make gcp-setup-docker

# 権限不足の場合
gcloud projects add-iam-policy-binding mlops-dev-a \
  --member="user:$(gcloud config get-value account)" \
  --role="roles/artifactregistry.writer"
```

### Cloud Run Job実行失敗（Secret Managerアクセスエラー）

```bash
make tf-apply
```

### Elastic Cloud接続タイムアウト

`deploy-all` 直後はElastic Cloudの起動に数分かかる場合がある。1〜2分待ってから再実行:

```bash
make ingest
```

### Terraformのstate不整合

```bash
make tf-import    # 既存リソースをstateにimport
make tf-plan      # 差分確認
```

### GCP APIが未有効

```bash
make gcp-setup-apis
```
